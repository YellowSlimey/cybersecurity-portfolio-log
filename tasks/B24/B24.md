## Design and Implement an Access Control System

### Description
For this task, I designed and implemented a simple **Role-Based Access Control (RBAC)** system to demonstrate how organisations manage user permissions and restrict access to sensitive actions.

The system simulates a basic environment where different user roles (Admin, Staff, and Guest) have different levels of access to resources. This reflects how real systems such as university portals, cloud platforms, and corporate systems enforce security policies.

### Evidence
**Code used:** `access_control_system.py`

### Evaluation
**Strengths**
- Simple and clear Role-Based Access Control model
- Easy to extend with new roles and permissions
- Effectively prevents unauthorized actions
- Reflects real-world access control principles used in enterprise systems

**Weaknesses**
- No user authentication (identity not verified)
- No logging or audit trail of access attempts
- Static permissions (not context-aware or dynamic)
- Not scalable for large distributed systems without modification

### Reflection
This task helped me understand how access control systems enforce security in real-world applications by restricting actions based on user roles. Implementing the system in Python showed how simple logical structures can effectively prevent unauthorized access. It also reinforced the importance of least privilege, where users only receive the permissions necessary for their role. Overall, this activity improved my understanding of fundamental cybersecurity access control principles and how they are applied in secure system design.
