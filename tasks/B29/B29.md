## CVE Analysis Using Three Generative AI Systems

### Description

I selected CVE-2025-26943, a SQL injection vulnerability in the WordPress Easy Quotes plugin, and analysed it using three generative AI systems: ChatGPT, Gemini, and Kimi. I provided each system with the same prompt asking for the cause of the vulnerability, how to fix it, and secure coding examples. I then compared the consistency and quality of their responses.

### Findings

All three AI systems correctly identified the vulnerability as a SQL injection issue caused by unsanitized user input and unsafe SQL query construction.

ChatGPT:
Provided a structured explanation of the vulnerability, clearly explained root causes such as unsafe string concatenation and missing prepared statements, and gave clean WordPress $wpdb->prepare() examples. The response was well-balanced and easy to understand.

Gemini:
Focused more on conceptual explanation and attack behavior (e.g., blind SQL injection and payload impact). It provided strong reasoning and developer practices but was slightly less structured in its coding examples compared to ChatGPT.

Kimi:
Gave the most technical and security-focused response, including CVSS score reference, layered defense strategies, and multiple secure coding patterns. It was the most detailed but slightly more complex to read.

Overall, all three systems recommended similar core fixes:

- use of prepared statements ($wpdb->prepare)
- input validation and sanitisation
- avoiding direct SQL concatenation

This activity showed me that while different AI systems provide similar cybersecurity advice, the depth, structure, and emphasis can vary significantly.
