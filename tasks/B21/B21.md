## Design and Implementation of a Role-Based Access Control (RBAC) System with Security Hardening

### Description
This project involved the design and implementation of a comprehensive Role-Based Access Control (RBAC) system from first principles, emphasizing cybersecurity best practices. The system was built in Python and simulates a real-world enterprise environment with hierarchical roles, granular permissions, session management, and defensive security mechanisms.

**System Architecture:**
The RBAC engine follows a three-tier architecture:

- Users — authenticated entities with attributes (active status, MFA, lockout state)
- Roles — permission containers supporting inheritance (e.g., Manager inherits from Staff)
- Resources — protected assets classified by type and sensitivity level (1–5)

**Key Security Features Implemented:**
- Granular Permissions: Five permission types (READ, WRITE, DELETE, EXECUTE, ADMIN, AUDIT) mapped to five resource types (Database, Files, API, Config, User Data)
- Role Inheritance: Hierarchical permission propagation with cycle detection
- Authentication & Session Management: SHA-256 password hashing, cryptographically secure session tokens (secrets.token_urlsafe), and 8-hour session expiry
- Account Lockout: Brute-force protection with a 3-attempt threshold and 30-minute lockout duration
- Comprehensive Audit Logging: Immutable event logging for all authentication and access decisions
- Resource Ownership: Automatic ownership-based access rights

**Test Environment:**
- 5 roles (Admin, Manager, Analyst, Staff, Auditor)
- 6 users with varying security postures
- 6 resources with sensitivity levels 1–5
